# 四大主流国内量化框架优劣分析（基于源码与架构维度）

## 1. VNPY (VeighNa)
- **官方链接**: https://github.com/vnpy/vnpy
- **架构特点**: 底层使用 C++ 封装各大交易接口（如国内期货的 CTP 接口），上层通过 pybind11 将接口暴露给 Python。系统核心采用了**事件驱动引擎（Event Engine）**，实现了数据流、日志记录、订单管理和策略模块的高度解耦。
- **优势**:
  - **生态最为完善**：国内开源量化的绝对霸主，社区庞大，文档和教程丰富。
  - **商品期货的实盘首选**：对 CTP（国内期货主力接口）支持极佳，历经市场多年检验，实盘极为稳定。
  - **高度模块化与扩展性**：源码结构极其清晰，按 `gateway`（网关接口层）、`app`（上层应用，如 CTA 策略引擎、组合策略引擎等）严格划分。非常适合深度阅读，并将其核心事件驱动思想**借鉴来搭建完全自主化的系统**。
  - **全链路打通**：支持从数据清洗、策略回测，到无缝切换至实盘交易。
- **劣势**:
  - 代码量庞大，学习曲线较陡峭。
  - 自带的 GUI 是基于 PyQt 编写的，对于习惯于 Web B/S 架构或想将其改造为轻量级纯后端服务的用户来说，剥离界面层有一定工作量。

## 2. 天勤量化 (TqSdk)
- **官方链接**: https://github.com/shinnytech/tqsdk-python
- **架构特点**: 一个纯 Python 实现的客户端 SDK。它的源码核心是通过 WebSocket/HTTP 直接连接信易科技（ShinnyTech）的云端服务器。大量的数据处理和撮合逻辑其实在云端完成，本地源码主要是 API Wrapper 和简单的事件轮询。
- **优势**:
  - **极简的 API 设计**：开发体验极佳。K线和 Tick 数据直接返回为 Pandas DataFrame，内置大量技术指标，几十行代码即可跑通一个策略。
  - **免维护数据源**：自带高质量的全品种历史行情数据，无需自己爬取和搭建本地数据库（对于基于分钟 K 线的开发极其友好）。
  - **平滑过渡实盘**：回测和实盘代码几乎完全一致，更改账户参数即可切换。
- **劣势**:
  - **强平台/云端绑定**：源码缺乏独立的本地核心撮合引擎深度控制权。高度依赖天勤的云端服务，如果云端异常，本地策略将受影响。
  - **商业化限制**：实盘交易和某些高级数据/特性需付费购买 Pro 版本，对于希望完全免费或深度二开的开发者存在局限。

## 3. 同花顺 (Financial-API)
- **官方链接**: https://github.com/HiThink-Tech/Financial-API
- **架构特点**: 这是一个偏向于金融数据获取和 AI Agent 交互的 API 框架。源码主要是数据抓取客户端、RESTful 接口封装以及针对现代大模型（LLM）MCP（Model Context Protocol）协议的对接层。
- **优势**:
  - **数据质量极高**：同花顺作为国内领先的金融信息服务商，其 API 提供的数据（尤其是股票行情、海量财务指标、精准复权数据）具有极高的权威性。
  - **面向 AI 与现代投研**：框架设计非常贴合当前的 AI Agent 潮流，方便将大模型分析能力与量化数据结合。
- **劣势**:
  - **定位偏向数据接口而非交易框架**：它更像是一个高质量的超级数据源（Data Feed），而不是一个像 VNPY 那样的完整“回测+实盘”事件驱动量化引擎。很难直接用它来管理资金仓位、订单生命周期和撮合。

## 4. 通达信框架 (总体评价，详细内容见 tdx_comparison.md)
- **架构特点**: 开源界多为逆向工程（如 PyTdx）获取数据的协议封装；官方则为高度封装的独立终端环境。
- **优势**: 股票市场占有率极高，数据传输极快（实时性好），官方接口极其稳定。
- **劣势**: 第三方开源工具通常缺乏安全的实盘交易接口（仅限读数据）；而官方量化平台又不够开放，难以将其核心剥离出来融入你**完全自主研发**的独立系统中。

## 💡 框架选型总结与推荐

针对您的核心需求：**“初期做商品量化（分钟级），未来计划横向扩展股票平台，考虑自主搭建或借用框架”**。

1. **若倾向于【基于现有框架进行二次开发】**：
   - **首选 VNPY**。VNPY 的 `cta_strategy` 模块就是为基于分钟 K 线的时间序列策略量身定制的。商品期货实盘的 CTP 接口在 VNPY 中最成熟。未来拓展股票时，VNPY 也有对应的网关支持。
2. **若倾向于【完全自主化搭建自己的系统】**：
   - **强烈建议深度剖析并借鉴 VNPY 和 Backtrader 的源码**。
   - 吸取 VNPY 的 `EventEngine`（事件引擎设计）和 `Gateway`（网关适配层设计），这代表了目前国内量化生产环境的最佳实践。
   - 数据源模块可以初期集成 **天勤 (TqSdk)**（用于期货数据获取）或 **通达信 API**（用于股票数据获取）来降低搭建本地数据库的门槛。

## 🌟 补充推荐：其他更适合的框架

除了您列出的这四个项目外，针对您的实盘交易场景和架构学习目标，强烈补充推荐以下三个主流框架：

### 1. QMT (迅投) / PTrade
- **官方链接**: QMT 属于商业级系统，主要通过各大券商发行（通常入金达到一定门槛免费开通）。开源社区有非常棒的 API 封装版本，如 **xtquant**：[https://github.com/skylot/xtquant](https://github.com/skylot/xtquant)
- **核心差异**: 如果说 VNPY 是“期货界”的王者，那么 QMT/PTrade 目前就是国内**“股票量化实盘交易”的绝对统治者**。它们由券商直接提供合规交易通道。
- **推荐理由**: 未来您的系统延伸到股票市场时，**实盘交易接口必须解决合规问题**，QMT 提供的 Python 接口（MiniQMT）是目前散户和私募做股票量化的最佳也是最主要的合规途径。

### 2. Backtrader
- **官方链接**: [https://github.com/mementum/backtrader](https://github.com/mementum/backtrader)
- **核心差异**: 这是一个极其纯粹的**策略回测框架**（并非原生针对实盘），由国外大神开发，采用 Python 纯面向对象设计。
- **推荐理由**: 如果您下定决心要**完全自主化搭建自己的量化交易系统**，那么 Backtrader 的源码是绝佳的“教科书”。它在数据源抽象（Data Feeds）、技术指标计算网络（Indicators）以及订单撮合生命周期的逻辑设计上，比 VNPY 更加抽象和优雅。非常适合借鉴其回测引擎的内核代码。

### 3. WonderTrader
- **官方链接**: [https://github.com/wondertrader/wondertrader](https://github.com/wondertrader/wondertrader)
- **核心差异**: 与 VNPY 类似，但更底层。采用了“C++ 核心交易引擎 + Python 策略接口”的架构，性能极高。
- **推荐理由**: VNPY 偏向于散户和中小私募，而 WonderTrader 架构上更贴近机构级别，支持更为复杂的底层数据管理、高频交易以及微服务部署。如果您对系统的极致性能（哪怕是分钟级但也追求绝对的低延迟处理）有要求，这套源码的借鉴价值极大。
